# Describing Software Development Process

## Impact of Software

Waterfall: Linear process
Agile: Frequent small incremental releases.
Prototyping: Build initial version to learn scope.
Rappid Application Prototyping: Incremental developed in parallel.

### Agile

Purpose: Which customer problems will the project solve. The five W's: what, why, who, where, when.
Process: Determine organizational processes.
People: People first

## Test-Driven Development

Write tests
Run these tests; they must fail
Write sources code
Run all tests; they must pass
Refactor the code where necessary

Gives a clear goal, make the test pass
Shows omitions before writing code
Uncovers edge cases before code starts
Make debugging faster, run the tests
Passing each test is a victory

## Code Review

Identify Bugs
Improve code quality
Get familiar with different parts of the project
Learn something new
